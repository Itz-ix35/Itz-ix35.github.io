B. A Plus B Problem

难度：★\bigstar★

题目大意：

有两个 nnn 位的可能含前导零的十进制整数 A,BA,BA,B，令 CCC 为它们的和的末 nnn 位，现在有 qqq 次操作，每次修改 AAA 或 BBB 中的某一位，你需要输出 CCC 中对应位的新的值，以及三个数中总共有多少数位发生了变化。

1≤n,q≤1061\leq n,q\leq 10^61≤n,q≤106

题目解法：

考虑如何维护进位：维护所有 A,BA,BA,B 对应位的和不为 999 的位，那么一位上有没有进位就取决于上一个和不为 999 的位的和是否 ≥10\ge 10≥10。

所以我们用 set 维护这些位，就可以 O(log⁡n)O(\log n)O(logn) 知道某一位当前是否有前面过来的进位，以及它的进位最多能影响后面多少位。

时间复杂度为 O((n+q)log⁡n)O((n+q)\log n)O((n+q)logn)。

D. Assumption is All You Need

难度：★★\bigstar\bigstar★★

题目大意：

给定 1,…,n1,\ldots,n1,…,n 的排列 A1,…,AnA_1,\ldots,A_nA1​,…,An​ 和 B1,…,BnB_1,\ldots,B_nB1​,…,Bn​，一次操作可以交换 AAA 中的一对逆序对，请判断能否通过若干次操作把 AAA 变成 BBB，如果可以还请构造。

1≤n≤20211\leq n\leq 20211≤n≤2021

题目解法：

考虑依次将 1,…,n1,\ldots,n1,…,n 归位。

比如先考虑 111，显然 111 只能往前交换，所以 111 交换之后不能到目标位置（它在 BBB 中位置）的前面，那么我们考虑 222，如果 222 在 AAA 中的位置在 111 之前，而又在 111 的目标位置之后，那么就可以交换 1,21,21,2。同时，如果在某组解中没有交换 1,21,21,2 而是交换了 1,x (x>2)1,x\ (x>2)1,x (x>2)，那么这一操作是可以用先交换 1,21,21,2，再交换 2,x2,x2,x 来代替的，所以先交换 1,21,21,2 肯定不会错过可行解（也可以理解成对于其他数来说 1,21,21,2 都一样，那么由于 111 还在目标位置左边，那么不如就把 222 尽量往后放）。

同理，再考虑 1,31,31,3，如果 333 又夹在 111 的当前位置和目标位置中间，就又可以交换 1,31,31,3，以此类推。

如果某一步中 xxx 当前位置已经在目标位置左边，那么无解。

时间复杂度为 O(n2)O(n^2)O(n2)。

E. Buy and Delete

难度：★\bigstar★

题目大意：

给定 nnn 个点 mmm 条边的边带权有向图 GGG，边权为正。

记一个子图 G′G'G′ 的权值 w(G′)w(G')w(G′) 为：最少可以将 G′G'G′ 中的边拆成多少个 DAG（空图权值为 000）。

你需要选择一组边权和不超过 CCC 的边集，使得其构成的子图权值最大，输出最大的权值。

1≤n≤2000, 1≤m≤50001\leq n\leq 2000,\ 1\leq m\leq 50001≤n≤2000, 1≤m≤5000

题目解法：

显然 w(G′)≤2w(G')\leq 2w(G′)≤2，因为可以分成 i<ji<ji<j 的边 (i,j)(i,j)(i,j) 和 i>ji>ji>j 的边 (i,j)(i,j)(i,j) 这两种。

而 w(G′)=0w(G')=0w(G′)=0 当且仅当没有边，w(G′)=1w(G')=1w(G′)=1 当且仅当 G′G'G′ 是 DAG。

所以使得 w(G′)≥1w(G')\ge 1w(G′)≥1 的最小边权和是边权的最小值，使得 w(G′)=2w(G')=2w(G′)=2 的最小边权和是最小环的边权和。

用 Dijkstra 求最小环即可。

时间复杂度为 O(nmlog⁡n)O(nm\log n)O(nmlogn)。

F. Illuminations II

难度：★★\bigstar\bigstar★★

题目大意：

给定一个凸 nnn 边形和一个凸 mmm 边形，后者完全包含在前者内部。

随机从 nnn 边形的内边界选择一个点，求从这个点能看到的 mmm 边形外边界的部分的长度期望。

3≤n,m≤2×1053\leq n,m\leq 2\times 10^53≤n,m≤2×105

题目解法：

从 nnn 边形上某个点能看到的 mmm 边形外边界的部分显然是一段连续的边。

根据期望的线性性，转化为求出 mmm 边形每条边能被 nnn 边形内边界上多长的一段点看到，只需要求出两个端点即可。

设 mmm 边形上一条边是 ABABAB，我们只需要求出直线 ABABAB 与 nnn 边形的两个交点即可，不妨考虑射线 ABABAB，为了快速对每条边求出这个射线与 nnn 边形的交点，可以用双指针转一圈，转化为只需要判定射线是否与 nnn 边形的某条边相交，而这是容易的。

时间复杂度为 O(n+m)O(n+m)O(n+m)。

H. Popcount Words

难度：★★★\bigstar\bigstar\bigstar★★★

题目大意：

定义 sis_isi​ 表示 popcount⁡(i) mod 2\operatorname{popcount}(i)\bmod 2popcount(i)mod2，令 w(l,r)w(l,r)w(l,r) 表示由 sl,…,srs_l,\ldots,s_rsl​,…,sr​ 依次拼接成的字符串。

给定 nnn 个区间 [li,ri][l_i,r_i][li​,ri​]，令 SSS 是由 w(l1,r1),…,w(ln,rn)w(l_1,r_1),\ldots,w(l_n,r_n)w(l1​,r1​),…,w(ln​,rn​) 依次拼接成的字符串。

还有 qqq 个询问，每个询问给定字符串 TTT，求 TTT 在 SSS 中的出现次数。

1≤n,q≤105, 1≤li,ri≤109, 1≤∑∣T∣≤5×1051\leq n,q\leq 10^5,\ 1\leq l_i,r_i\leq 10^9,\ 1\leq \sum |T|\leq 5\times 10^51≤n,q≤105, 1≤li​,ri​≤109, 1≤∑∣T∣≤5×105

题目解法：

这里的 w(l,r)w(l,r)w(l,r) 是一个很经典的递归构造的字符串，令 S(0,0)=0, S(0,1)=1S(0,0)=\texttt{0},\ S(0,1)=\texttt{1}S(0,0)=0, S(0,1)=1，然后 S(i+1,0)=S(i,0)+S(i,1), S(i+1,1)=S(i,1)+S(i,0)S(i+1,0)=S(i,0)+S(i,1),\ S(i+1,1)=S(i,1)+S(i,0)S(i+1,0)=S(i,0)+S(i,1), S(i+1,1)=S(i,1)+S(i,0)，那么 w(l,r)w(l,r)w(l,r) 就可以拆分成 O(log⁡r)O(\log r)O(logr) 个 S(i,j)S(i,j)S(i,j) 的拼接。

所以 SSS 可以拆分成 O(nlog⁡V)O(n\log V)O(nlogV) 个 S(i,j)S(i,j)S(i,j) 的拼接，其中 VVV 是 li,ril_i,r_ili​,ri​ 的值域。

建出所有模式串的 AC 自动机，那么我们只需要先预处理出所有 S(i,j)S(i,j)S(i,j)（只有 O(log⁡V)O(\log V)O(logV) 个）在 AC 自动机上运行的结果（从一个点会跳到哪个点），就可以算出 SSS 在 AC 自动机上行进的整个过程，而由于 S(i,j)S(i,j)S(i,j) 是递归定义的，所以我们可以用类似倍增的方法算出所有 f(u,i,j)f(u,i,j)f(u,i,j) 表示从 AC 自动机上点 uuu 出发走 S(i,j)S(i,j)S(i,j) 这个串会走到哪里。

然后我们要统计答案，先算出 AC 自动机上每个点经过了几次：由于我们已经可以知道从某个点 uuu 出发走某个 S(i,j)S(i,j)S(i,j) 走了几次（记为 g(u,i,j)g(u,i,j)g(u,i,j)），所以我们可以将这个过程再次根据递归结构“分解”，也就是从 g(u,i,j)g(u,i,j)g(u,i,j) 递推到 g(u,0,0)g(u,0,0)g(u,0,0) 和 g(u,0,1)g(u,0,1)g(u,0,1)，也就知道每个点走了几次了。

然后再 Fail 树上 DP 一下得到每个串的匹配次数。

时间复杂度为 O((n+∑∣T∣)log⁡V)O((n+\sum |T|)\log V)O((n+∑∣T∣)logV)。

I. PTSD

难度：★\bigstar★

题目大意：

给定长度为 nnn 的 0,1\texttt{0,1}0,1 串 SSS，你要将 {1,…,n}\{1,\ldots,n\}{1,…,n} 划分成若干个子集，使得所有满足 Si=1S_i=\texttt{1}Si​=1 且 iii 在其所属子集中为次大值的 iii 之和最大。

1≤n≤1061\leq n\leq 10^61≤n≤106

题目解法：

从大到小贪心，维护当前还剩几个可以作为最大值的数，如果还有这样的数且当前位为 111，就将它们分入一个集合，否则就将当前数也加入可以作为最大值的候选。

时间复杂度为 O(n)O(n)O(n)。

J. Suffix Automaton

难度：★★\bigstar\bigstar★★

题目大意：

给定字符串 SSS，将所有 SSS 的本质不同非空子串按长度为第一关键字，字典序为第二关键字排序，qqq 次询问其中第 kkk 个子串是什么（输出最前一次出现的左右端点）。

1≤∣S∣,q≤1061\leq |S|,q\leq 10^61≤∣S∣,q≤106

题目解法：

本题用后缀数组处理更为直观一些。

预处理后缀数组，考虑将 height 数组按值排序，离线询问，三者（询问值，height 值和子串长度）同时从小到大考虑。

假设当前考虑到了长度为 iii 的子串，我们维护长度小于 iii 的子串数量，还维护所有长度为 iii 的不同子串在后缀数组中的首次出现位置，于是我们就只需要快速求出其中的第 kkk 小，用一个维护第 kkk 小值的倍增树状数组即可，而求出最前一次出现的左右端点只需要利用一个建立在后缀数组上的查询最小值的 ST 表即可实现。

时间复杂度为 O((∣S∣+q)log⁡∣S∣)O((|S|+q)\log |S|)O((∣S∣+q)log∣S∣)。

对于每个 i∈[2,n]i\in [2,n]i∈[2,n]，求出从 111 到 iii 走一条最短路，至少需要支付多少钱。
