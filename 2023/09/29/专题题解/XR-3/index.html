A. 等差数列

∣a1∣,∣a2∣,n≤106|a_1|,|a_2|,n\leq 10^6∣a1​∣,∣a2​∣,n≤106

答案为 (n2)×(a2−a1)+na1\binom n 2\times (a_2-a_1)+na_1(2n​)×(a2​−a1​)+na1​，时间复杂度 O(1)O(1)O(1)。

B. 小道消息

难度：★\bigstar★

题目大意：

有 nnn 个人，代号为 2,…,n+12,\ldots,n+12,…,n+1，第零秒将一个信息告诉第 kkk 个人，每秒内，知道消息的人会将消息告诉所有代号和他的代号互素的人，问至少多少秒后所有人都知道了信息。

2≤n≤10142\leq n\leq 10^{14}2≤n≤1014

题目解法：

众所周知 (n,2n](n,2n](n,2n] 中有至少一个素数，所以答案至多是 222，只需要判定答案是不是 111，即 k+1k+1k+1 是否与所有其他数互素。

时间复杂度为 O(n)O(\sqrt n)O(n​)。

C. 核心城市

难度：★\bigstar★

题目大意：

给定 nnn 个点的树，你可以选择包含 kkk 个点的一个连通块，最小化所有点到连通块的最短距离的最大值，只需要输出这个最小的最大值。

n≤105n\leq 10^5n≤105

题目解法：

二分答案 CCC，考虑枚举连通块的 LCA，LCA 之下只要是到子树内最远点距离不小于 CCC 的点就要选，直接进行一个换根 DP 就可以了。

时间复杂度为 O(nlog⁡n)O(n\log n)O(nlogn)。

事实上也可以先证明直径中点必选，然后直接排序选择，可以 O(n)O(n)O(n)。

D. 系统设计

难度：★★\bigstar\bigstar★★（更优复杂度 ★★★\bigstar\bigstar\bigstar★★★）

题目大意：

给定 nnn 个点的有根树，长度为 mmm 的序列 a1,…,ama_1,\ldots,a_ma1​,…,am​，支持 qqq 个操作：

  • 修改操作：序列的单点修改；
  • 查询操作：给定 x,l,rx,l,rx,l,r，求从 xxx 出发，依次考虑 i=al,…,ari=a_l,\ldots,a_ri=al​,…,ar​，如果 xxx 有大于 iii 个儿子就令 xxx 变为其编号第 iii 小的儿子，否则终止过程，问过程结束后 xxx 到了哪里。

1≤n,m,q≤5×1051\leq n,m,q\leq 5\times 10^51≤n,m,q≤5×105

题目解法：

考虑重链剖分，查询时先快速跳掉在重链上下行的部分，然后走一条轻边，由于轻边最多只会走 log⁡n\log nlogn 次，所以只需要快速求出在一条重链上能走多远，而这只需要预处理重链上每个点重儿子编号相对大小构成的序列的区间 Hash 值，与序列 aaa 的区间 Hash 值比对即可。

具体地，由于存在修改，可以用线段树维护 aaa 的区间 Hash，查询时直接二分则总复杂度为 O(nlog⁡3n)O(n\log^3 n)O(nlog3n)，如果线段树上二分则总复杂度为 O(nlog⁡2n)O(n\log^2 n)O(nlog2n)。

然而还有一种更好的做法，首先我们可以“回退”根结点到 xxx 的这一段（视为在 al,…,ara_l,\ldots,a_ral​,…,ar​ 这个序列前加入一些数），将问题转化为从根结点出发，那么只需要预先存好每个点对应的从根结点出发的路径的 Hash 值，就只要查询最大的 mmm 使得 al,…,ama_l,\ldots,a_mal​,…,am​ 的 Hash 值与某个点的 Hash 值匹配，采用线段树上二分，复杂度为 O(nlog⁡n)O(n\log n)O(nlogn)。

E. Namid[A]me

难度：★★★\bigstar\bigstar\bigstar★★★

题目大意：

给定 nnn 个点的点带权树，设其叶子（度数为 111 的点）个数为 ddd，点 iii 的权值为 aia_iai​，令 f(u,v)f(u,v)f(u,v) 为 u→vu\to vu→v 路径上点权与，对所有无序对 (u,v)(u,v)(u,v) 求和 f(u,v)f(u,v)f(u,v)^{f(u,v)}f(u,v)f(u,v)，答案对 786433786433786433 取模，模数有一个原根是 101010。

1≤n≤2×105,  ai<230,  1≤n×d≤3×1061\leq n\leq 2\times 10^5,\ \ a_i<2^{30},\ \ 1\leq n\times d\leq 3\times 10^61≤n≤2×105,  ai​<230,  1≤n×d≤3×106

题目解法：

我的做法似乎和题解不太一样。

显然树中的任意一条路径是以某个叶子为根时的一段祖先链，我们不妨依次枚举叶子 l1,…,ldl_1,\ldots,l_dl1​,…,ld​ 为根，统计祖先链的贡献。

对于确定的 uuu，枚举其祖先 vvv 的过程中，显然 f(u,v)f(u,v)f(u,v) 只有不超过 log⁡A\log AlogA 种取值（AAA 是 aia_iai​ 的上界），并且在 DFS 过程中对每一位记录最深的这一位为 000 的点就可以得到这 log⁡A\log AlogA 种值和它们的分界点，于是只需要 O(nlog⁡A)O(n\log A)O(nlogA) 就可以统计所有祖先链的答案。

但是这样会重复计算，所以我们不妨让 lil_ili​ 为根时不再计算在 l1,…,li−1l_1,\ldots,l_{i-1}l1​,…,li−1​ 为根时已经计算过的那些路径。

设祖先链是 u→vu\to vu→v（vvv 是祖先，uuu 是后代），再设 www 是 vvv 到 uuu 路径上 vvv 的后继点，那么这条祖先链之前没有被算过，当且仅当 l1,…,li−1l_1,\ldots,l_{i-1}l1​,…,li−1​ 都在 www 的子树中，且都不在 uuu 的子树中，也就是说我们只是对 uuu 的选取进行限制，然后再限制 vvv 是 uuu 的深度不超过一个定值的祖先，还是可以用与上面相同的 DFS 方法求解。

注意利用原根使得幂的计算变为 O(1)O(1)O(1) 的。

时间复杂度为 O(dnlog⁡A)O(dn\log A)O(dnlogA)。

F. Unknown Mother-Goose

难度：★★★★\bigstar\bigstar\bigstar\bigstar★★★★

题目大意：

给定 nnn 和一个正整数集合 SSS，求 xxx 的数量，使得：

  • 3≤x≤n3\leq x\leq n3≤x≤n；
  • 存在 a∈Sa\in Sa∈S 使得 a∣xa\mid xa∣x；
  • 存在 b∈Sb\in Sb∈S 使得 b∣(x−1)b\mid (x-1)b∣(x−1)；
  • 存在 c∈Sc\in Sc∈S 使得 c∣(x−2)c\mid (x-2)c∣(x−2)。

3≤n≤109,  ∣S∣≤203\leq n\leq 10^9,\ \ |S|\leq 203≤n≤109,  ∣S∣≤20

题目解法：

考虑暴力，复杂度是 O(n+∑x∈Snx)O(n+\sum\limits_{x\in S}\dfrac{n}{x})O(n+x∈S∑​xn​)，难以通过。

使用位运算来加速这一过程，设一个长度为 n+1n+1n+1 的 01\texttt{01}01 串 TTT 满足：TiT_iTi​ 为 1\texttt{1}1 当且仅当 i−1i-1i−1 是某个 x∈Sx\in Sx∈S 的倍数。

我们要查询的是 TTT 中有多少个连续三个 1\texttt{1}1，如果压位存储，只需要类似 popcount(T|(T>>1)|(T>>2)) 的运算就可以求出答案了。

用 unsigned long long 存储 TTT，但我们还是没有减小这个 nx\dfrac{n}{x}xn​ 的复杂度，因此还要对 xxx 分类讨论：

  • 如果 x≥64x\ge 64x≥64，那么直接采用 nx\dfrac{n}{x}xn​ 的暴力算法；
  • 如果 x<64x<64x<64，那么 xxx 的倍数对应的对 TTT 的影响是周期的，有一个周期是 64x64x64x，而 646464 恰好又是 unsigned long long 的位数，我们只要用一个辅助数组存储 xxx 对 TTT 的前 xxx 个 unsigned long long 单元的影响，然后周期性地将它的第 iii 个单元或到 TTT 的第 jx+ijx+ijx+i 个单元即可，利用位运算，可以做到复杂度 O(n64+64)O(\dfrac{n}{64}+64)O(64n​+64)。

时间复杂度为 O(n∣S∣w)O(\dfrac{n|S|}{w})O(wn∣S∣​)，其中 w=64w=64w=64。
